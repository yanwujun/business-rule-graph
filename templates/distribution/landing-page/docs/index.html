Eight guides

Pick where you are. Each guide stands alone; together they take you from pip install to a CI pipeline that gates on structural review.

1. Getting Started Install, index your repo, ask the first structural question, run a pre-merge check, plan parallel agent work, sign a Code Graph Attestation. ~10 minutes start to finish. 2. Integration Tutorials End-to-end MCP setup for Claude Code, Cursor, Gemini CLI, Codex CLI, and Amp (Sourcegraph). Per-platform commands and the cross-platform validation checklist. 3. Command Reference All 211 commands, grouped by intent: exploration, health, refactoring, security, governance. Each entry has flags, output, and a concrete usage example. 4. Architecture How the index pipeline, storage, graph engines, retrieval, and patch verifier fit together. The system diagram + subsystem responsibilities + command-to-data flow. 5. The Agent Contract Five rules an AI coding agent should follow when Roam is available — context before edit, impact before delete, critique before merge, simulate before refactor, math before optimise. The discipline that turns capabilities into outcomes. 6. How Roam thinks The decision tree. Nine moments in an engineering workflow, mapped to the four-to-five Roam commands that answer each. Match the question to the command, not the other way around. 7. What AI misses, Roam catches Eight concrete scenarios — clones, blast radius, accidental O(n²), N+1 queries, layer violations, missing tests, refactor safety, signed audit evidence — each with the exact command output Roam produces. Read this to feel what the tool does. 8. Troubleshooting Eight things that go wrong with Roam and the one command that usually fixes each — stale indexes, cloud-sync conflicts, missing extras, parser failures, MCP setup, cache permissions, OOM during indexing, slow JSON output.
